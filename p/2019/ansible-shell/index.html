nfs 服务器的迁移

common.sh 和 nfs.sh 脚本位置
首先配置要使用 nfs 服务的服务器路径

# /etc/exports 文件
/home/www/某个站点 ip地址(rw,sync,all_squash,no_subtree_check)
/home/www/某个站点 IP地址(rw,sync,all_squash,no_subtree_check)

/home 路径是下图 mounted on 下的路径

参数说明:

  • rw 读写权限
  • sync 数据同步写入内存缓冲区和硬盘, 保证数据一致
  • all_squash 所有访问用户都映射为匿名用户或用户组
  • no_subtree_check 即使输出目录是一个子目录，nfs服务器也不检查其父目录的权限

或者直接

然后再重启一下就行了

如果用的是 firewalld

我搭建的 rsync 是这样子的，主服务器 (也就是正在运行Web服务器的服务器) 文件发生改变 (内容改变或者文件本身的增加或者删除) 就会通过监听脚本自动备份到备份服务器

以下安装和配置 rsync 是参考该链接 站长那些事儿

两台服务器一起安装 rsync

还要增加一个配置文件 (这个配置文件在CentOS7下不存在，我们手动创建)

内容如下

# default: off
# description: The rsync server is a good addition to an ftp server, as it \
#       allows crc checksumming etc.
service rsync
{
        disable = no
        flags           = IPv6
        socket_type     = stream
        wait            = no
        user            = root
        server          = /usr/bin/rsync
        server_args     = --daemon
        log_on_failure  += USERID
}

是 从这里 找滴，亲测可用

接下来是 负责备份的服务器 的事情了

添加以下内容

log file = /var/log/rsyncd.log
pidfile = /var/run/rsyncd.pid
lock file = /var/run/rsync.lock
secrets file = /etc/rsync.pass
motd file = /etc/rsyncd.Motd
#模块名称
[test]
#你想备份在本服务器哪个路径就填呗
path = /home/test
#描述随便加
comment = just a test
uid = 0
gid = 0
port = 873
use chroot = no
read only = no
list = no
max connections = 200
timeout = 600
#认证用户名，可以随便取
auth users = test_user
#主服务器IP地址
hosts allow = 你的ip地址

认证用户和密码文件，系统原本没有，需要我们手动创建的喔。
该文件名字可以随意

里面是内容是 键值对

test_user:testpass

这里注意啦，必须是权限必须是 600 喔

启动 rsync 有些需要有些不需要， 可能是防火墙原因阻拦了其他端口…
